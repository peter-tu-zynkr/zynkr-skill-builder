# 文章大綱型態（AEO-first）

對應流程圖「文章大綱」。設計大綱時的型態與規則。

## 鐵則：開頭即答（answer-first）

第一段（80–120 字內）直接回答標題問句，讓 AI 引擎能整段引用。其餘段落再展開。

## 常見大綱型態（依意圖選）

| 意圖 | 建議型態 | 骨架 |
|---|---|---|
| 資訊型 | 概念→分類→應用 | 定義 → 為什麼重要 → 怎麼做（步驟）→ 案例 |
| 商業調查 | 比較 / 清單 | 選項總覽 → 比較表 → 各選項拆解 → 怎麼選 |
| 交易決策 | 案例 / 決策框架 | 痛點 → 方法 → 真實案例/ROI → 怎麼開始（CTA） |

## 每個 H2 要標

- 一個要點方向（不只是標題）
- 目標 / 輔助關鍵字落點
- 3 個可寫要點（給 article-drafter）

## AEO 結構元件（按需放入大綱）

- FAQ 區塊（3–5 題，schema 用）
- 比較表（商業調查意圖必備）
- 步驟 / 清單（可被擷取為 list）
- 一個「命名框架」（讓內容有可被引用的記憶點）

## 銜接既有寫作管線

交棒摘要的「選定架構 + 各章節 3 要點」格式，必須與 `article-style-selector` 一致，讓 `article-drafter` 無痛接手。額外欄位（關鍵字落點 / FAQ / AEO / CTA）附在後面即可。
