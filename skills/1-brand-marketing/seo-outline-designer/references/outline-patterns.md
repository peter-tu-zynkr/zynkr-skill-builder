# 文章大綱型態（decision-first + AEO）

對應流程圖「文章大綱」。設計大綱時的型態與規則。聲音以 Zynkr Brand Guide（decision-first）為準。

## 鐵則一：開頭即答（answer-first，給 AEO）

第一段（80–120 字內）直接回答標題問句，讓 AI 引擎能整段引用。

## 鐵則二：Method 現形（給品牌）

大綱至少讓 Zynkr Method 一個 move 看得見：Frame（先框定真正的決策）/ Clarify / Constrain / Compare（點出 trade-off）/ Commit（收在明確方向）。決策型文章建議走「框定決策 → 真實取捨 → 給方向」的骨架。

## 常見大綱型態（依意圖選）

| 意圖 | 建議型態 | 骨架 |
|---|---|---|
| 資訊型 | 概念→分類→應用 | 定義 → 為什麼重要 → 怎麼做 → 案例 |
| 商業調查 | 比較 / 取捨 | 選項總覽 → 真實 trade-off 比較表 → 各選項拆解 → 怎麼選 |
| 交易決策 | 決策框架 | 你在做的決策 → 背後的取捨 → 第一手經驗佐證 → 明確方向（CTA） |

## 每個 H2 要標

- 一個要點方向（不只是標題）
- 目標 / 輔助關鍵字落點
- 3 個可寫要點（給 article-drafter）

## AEO 結構元件（按需放入）

- FAQ 區塊（3–5 題，schema 用）
- 比較表（商業調查意圖必備；用來呈現 trade-off）
- 步驟 / 清單（可被擷取為 list）
- 一個「命名框架」——最好是一個決策框架 / trade-off 的命名

## 銜接既有寫作管線

交棒摘要的「選定架構 + 各章節 3 要點」格式須與 article-style-selector 一致，讓 article-drafter 無痛接手；額外欄位（關鍵字落點 / FAQ / 收尾方向 / AEO）附在後面。
