# SEO 撰寫 Brief 範本（decision-first）

對應流程圖綠色節點「Brief 格式」。每篇文章一份。聲音以 Zynkr Brand Guide 為準。

```
# SEO Brief：<文章工作標題>

## 基本
- 目標關鍵字（主）：
- 輔助關鍵字：
- 搜尋意圖：資訊 / 商業調查 / 交易決策
- 漏斗階段：TOFU / MOFU / BOFU
- 內容支柱：<五大 decision-first 支柱之一>
- 語言：zh-TW（旗艦 EN：是 / 否）

## 讀者與決策
- 目標人物誌：<來自 Persona 包>
- 讀者正面對的決策（decision state）：<他卡在什麼選擇>
- 讀完後能做出的更好決策（desired outcome）：

## 角度與勝出（decision-first）
- 要逼讀者面對的 sharper question：
- 真實的 trade-off（不是 A vs B，而是 ___ vs ___）：
- 價值主張（一句）：
- 勝出關鍵（2–3 點）：

## 內容
- 必涵蓋重點（bullet）：
- 第一手經驗 / 案例 / 數據（作為支撐判斷的證據）：（不足標 [需 Peter 補]）
- 必答 FAQ（給 AEO）：3–5 題
- 收尾的明確方向（Commit）：讀者明天能做的下一步

## 轉換
- 主要 CTA：<B2B：預約決策診斷 / discovery call；或 lead magnet>
- 內部連結對象：<支柱頁 / 相關文章 / /consult>

## 聲音檢查
- Zynkr Method 至少一個 move 現形：Frame / Clarify / Constrain / Compare / Commit
- 善用詞：決策 · 取捨 · 判斷 · 方向 · 脈絡 · 顧問夥伴
- 禁用詞（不可出現）：賦能 / empower · 生產力工具 / productivity tool · AI-powered · 無縫 / seamless · 智慧助理 · 顛覆 · supercharge

## AEO 要求
- 開頭即答（answer-first）一句：
- 結構化區塊：FAQ / 比較表 / 步驟 / schema
- 一個可被引用的「命名框架」（最好是一個決策框架 / trade-off 的命名）：
```

---
欄位填不滿就標記，交棒前經人工「驗證並補充」。Brief 是大綱與寫作的唯一真實來源。
