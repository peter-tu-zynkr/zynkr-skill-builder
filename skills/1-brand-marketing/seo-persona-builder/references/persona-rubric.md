# 消費者人物誌 Rubric（B2B 優先）

對應 v2 流程圖綠色節點「消費者輪廓和人物誌」。每個人物誌填滿以下欄位才算完整。

## 必填欄位

| 欄位 | 說明 |
|---|---|
| 角色與職稱 | 例：某中型企業 L&D 經理 / 營運轉型負責人 |
| 組織情境 | 公司規模、產業、AI 導入成熟度 |
| 核心痛點 | 對照三鴻溝：知識（找不到答案）／認知（不知怎麼解）／執行（不會落地） |
| 目前解法 | 現在用什麼方式處理（外包、內部試、不處理） |
| 購買旅程階段 | 認知 TOFU / 評估 MOFU / 決策 BOFU |
| 搜尋語言 | 這個人會在 Google / ChatGPT 打什麼字（口語、非關鍵字工具語言） |
| 反對意見 | 為什麼還沒行動（預算、風險、沒人用） |
| 成功樣貌 | 導入後對他個人 KPI 的意義 |

## 主要 vs 次要受眾

- **主要（漏斗目標 = B2B discovery call）**：HR、L&D、營運、數位轉型主管。
- **次要（暖身 + 社會證明）**：想提升 AI 素養的專業工作者（B2C）。

## 品質檢查

- 痛點是否具體到可以反推出搜尋語句？（不能只寫「想用 AI」）
- 是否標出此人最可能問 AI 引擎的一句話？（給後面 AEO 用）
- 三個人物誌之間是否 MECE，不重疊？
