## Agent Specs

### 1. Sales & Intake Agent

**Purpose:** Qualify inbound/outbound leads and capture structured intake data.

**Input:**
- Lead source (inbound form, outbound prospecting, referral)
- Initial contact notes

**Process:**
- Run qualification checklist (problem fit, budget signal, timeline, decision-maker)
- Guide intake conversation with structured questions
- Summarize scoping context

**Output:**
- Qualified/disqualified verdict with rationale
- Completed intake form
- Initial scope summary (problem statement, estimated complexity, suggested next step)

**Node type:** Conversational | Q-A | Human-driven

---

### 2. Discovery Agent

**Purpose:** Map the as-is process and surface friction points.

**Input:**
- Intake form output (from Agent 1)
- Process owner interview notes / SOPs / tool logs

**Process:**
- Generate structured interview questions by role
- Identify friction points: high-traffic, high-friction, high-impact steps
- Compare as-documented vs. actual observed flow
- Produce SIPOC diagram (Suppliers, Inputs, Process, Outputs, Customers)

**Output:**
- Pain Point Heatmap (step x friction type matrix)
- Ground-truth as-is process map
- SIPOC diagram (text format → export to Lucidchart)

**Node type:** Conversational | RAG | Human-driven

---

### 3. Design Agent

**Purpose:** Redesign the workflow and assess automation feasibility.

**Input:**
- Discovery outputs: Pain Point Heatmap + as-is process map + SIPOC

**Process:**
- Apply FE / BE / DB swimlane structure to redesigned flow
- Score Digital Readiness (5 dimensions, 1-3 scale, 15-point max)
- Score Workflow Selection (4 dimensions, 1-3 scale, 12-point max)
- Apply I-P-O validation to each step
- Identify modular, reusable logic blocks

**Output:**
- FE/BE/DB swimlane diagram (Lucidchart-ready spec)
- Digital Readiness Scorecard (score + recommendation)
- Workflow Selection Scorecard (score + recommendation)
- Modular process library (labeled blocks with I-P-O contracts)

**Node type:** Context-Direction | Contextual | Deductive | AI-driven

---

### 4. Proposal Agent

**Purpose:** Package the redesigned solution into a client-ready proposal.

**Input:**
- Design outputs: swimlane diagram, scorecards, modular process library
- SIPOC (from Discovery)

**Process:**
- Validate solution against SIPOC scope boundaries
- Draft solution architecture narrative (FE/BE/DB per stage)
- Define scope, deliverables, and exclusions
- Flag risks and open questions

**Output:**
- Solution architecture document
- Scope & deliverables table
- Proposal deck outline (section headers + key points per section)

**Node type:** Instruction-Execution | Contextual | Deductive | AI-driven

---

### 5. Development Agent

**Purpose:** Build the AI assistant node bundle for each workflow step.

**Input:**
- Approved proposal + swimlane diagram
- Node position in workflow (FE / BE / DB ownership)

**Process:**
- Assign 5D routine tag (Why / What / How / Who / When) per node
- Check for known 5D conflicts (e.g., Contextual + Transactional)
- Draft 3-Layer Fold prompt (A: Control Layer / B: Node Logic / C: Run Payload)
- Define handoff payload spec (verified facts, decisions, open questions, status flags)
- Generate 3 test cases per node (normal / edge / missing input)

**Output (per node):**
- Node contract sheet (Input / Process / Output)
- 5D routine tag
- Folded prompt package (Layer A + B + C)
- Handoff payload spec
- 3 test cases + pass/fail results

**Node type:** Instruction-Execution | RAG + Tool-based | Deductive | AI-driven | Transactional

---

### 6. Deploy Agent

**Purpose:** Coordinate production rollout and user onboarding.

**Input:**
- Completed node bundles (from Agent 5)
- Deployment environment details (platform, permissions, user list)

**Process:**
- Run UAT checklist against each node's acceptance criteria
- Generate staged rollout plan (pilot group → full rollout)
- Produce onboarding guide (role-specific, step-by-step)
- Draft change management messaging

**Output:**
- UAT results report (pass / fail / remediation notes)
- Staged rollout plan
- Onboarding guide
- Change management communication templates

**Node type:** Instruction-Execution | Contextual | Human-driven | Transactional

---

### 7. Govern Agent

**Purpose:** Keep deployed agents running, current, and discoverable.

**Input:**
- Deployed agent bundle + onboarding guide
- Usage logs, feedback, ratings

**Process:**
- Monitor knowledge freshness (flag outdated sources)
- Triage feedback: bug / knowledge gap / scope change
- Generate AI Marketplace listing entry (name, IPO, category, setup steps)
- Recommend iterative improvements per feedback pattern

**Output:**
- Knowledge audit report (current / stale / missing)
- Feedback triage log (category + priority)
- AI Marketplace listing (ready to paste into index sheet)
- Improvement backlog (prioritized list of changes)

**Node type:** Cyclical | RAG | Deductive | AI-driven | Cyclical

---

## Build Order

Build sequentially — each agent's output feeds the next:

```
1. Sales & Intake → 2. Discovery → 3. Design → 4. Proposal → 5. Development → 6. Deploy → 7. Govern
```

Start with Agent 5 (Development) for internal use, since it directly accelerates the highest-volume work.
