# Process Livestream

Orchestrate four parallel agents to transform a livestream transcript into reusable course content, a knowledge base update, a published recap document, and new content ideas for the marketing pipeline.

## Step 1a — Transcript intake (always runs)

The transcript file was already loaded into context by the user (via `@file` reference). This step only handles renaming and path storage — no file reading required.

**Compute standardized filename:**
1. Take `stream_title`, convert to lowercase
2. Replace spaces and special characters with hyphens; remove consecutive hyphens
3. Append `-[stream_date].cleaned.md`
4. Format: `[slug]-[YYYY-MM-DD].cleaned.md`

**Determine what action is needed:**

- If the source file is already in `[STREAM_DIR]/video-recording/` **and** its basename matches the standardized filename → no action needed; store the path as-is.
- If the source file is in `[STREAM_DIR]/video-recording/` but has a different name → rename in place:
  ```bash
  mv "[source_file_path]" "[STREAM_DIR]/video-recording/[standardized_filename]"
  ```
- If the source file is outside `[STREAM_DIR]/video-recording/` → copy it in (do not delete the original):
  ```bash
  cp "[source_file_path]" "[STREAM_DIR]/video-recording/[standardized_filename]"
  ```

Store the resulting absolute path as `FILE_PATH` and the filename as `FILE_NAME` for use in Step 10.

If the rename or copy fails, display the error and stop.

Display:
```
✅ 逐字稿就緒
   原始檔案：[source_file_path]
   標準化名稱：[FILE_NAME]
```

**Sync procedure** (applies to Gates A, B, C, D):
1. Write approved content to the local file first.
2. Read the full updated local file.
3. Use `mcp__google-workspace__get_doc_content` to get the current Google Doc content.
4. Use `mcp__google-workspace__batch_update_doc` to replace the Google Doc body with the full updated local file content (delete existing text from index 1 to end, then insert at index 1).

6. If any MCP sync call fails, report the error but do not block the workflow — the local file is the source of truth.

   ```python
   from googleapiclient.http import MediaFileUpload
   media = MediaFileUpload(xlsx_path,
       mimetype='application/vnd.openxmlformats-officedocument.spreadsheetml.sheet',
       resumable=True)
   file_meta = {
       'name': '[活動名稱]_[YYYY.M.DD]',
       'mimeType': 'application/vnd.google-apps.spreadsheet',
       'parents': ['[roster_folder_id]']
   }

   ```

## Error Handling

- If the admin agent returns `blocked` or `error` during transcript intake, stop the entire workflow — there is no transcript to process.
- If Agent 1 (Event Recap) returns an error, skip Gate E and note it in the final summary.
- If Agent 2 returns no Q&A pairs, skip Gate A entirely.
- If Agent 3 returns no qualifying segments, skip Gate B. Still run Gate C if unmatched segments exist.
- If a chapter file write fails, report the specific file and continue with others.
- If Agent 4 returns no qualifying ideas, skip Gate D entirely.
- If `_matrix.md` cannot be written, report the error — do not attempt to write `_sourcing-log.md` either.
- If any MCP call in Gate E fails, show the error and offer to retry or skip.
- Never silently swallow errors — always surface them in the status board.
