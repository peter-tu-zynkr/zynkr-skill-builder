# Frontend

This is the Next.js frontend for the Zynkr skill directory.

## Local Development

Run the app locally from this folder:

```bash
npm run dev
```

The local dev server runs at:

```text
http://localhost:3000
```

For a production verification build:

```bash
npm run build
```

## Deployment

This frontend is intended to be deployed on Zeabur, not Vercel.

Project deployment notes live in:

- `../deploy/deploy-plan.md`

Current Zeabur expectation:

- service root directory: `frontend/`
- build command: `npm run build`

## Notes

- The frontend reads normalized data from `frontend/lib/generated-skills.json`.
- IPO fields shown in the UI are generated by `scripts/ingest.ts`, not parsed in the frontend.
- If IPO content looks wrong, fix the ingest transform or the source data, then regenerate artifacts.
- Legacy CSV parity is checked outside the frontend with `cd ../scripts && npm run check-ipo`.
- The target architecture is repo-managed content plus generated artifacts.
